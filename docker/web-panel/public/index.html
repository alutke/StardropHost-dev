Backups

Backups are full snapshots of your entire game config — saves, preferences, and options. They are taken automatically every 24 hours and before every server stop, restart, or update. Use Backup Now before making any risky changes. To recover from a crash or bad state, hit Restore on any backup — the server will stop and you will need to restart it after. Upload Backup accepts a previously downloaded .zip backup to restore.

              playit.gg Setup Guide
              RECOMMENDED
              Paid / Free — No client setup needed
            

            

              
    
                
  • No client setup needed — friends join directly with an address
    • 
                
  • Multiplatform — PC, Mobile, Console (Console untested)
    • 
              

              

                Premium is highly recommended. Free tier works but has no TCP —
                StardropHost/Stardew works on UDP (PC & Mobile only). Consoles require TCP/UDP.
                Remote Dashboard access requires a TCP tunnel (Premium only).
              

              
    
                
  • Paid is only ~$5/mo — worth it for custom domain names and other premium features
    • 
                
  • Agents + Tunnels are essential for multi-instance Stardew / StardropHost installs
    • 
                
  • Supports firewall and proxy setup for safe, monitorable connections
    • 
              

            


            
            

              Account Setup
              
*Steps 2, 3 and 4 are for Premium accounts only

              
    
                
  1. Sign up for an account at playit.gg
    2. 
                
  2. Don't set up an agent yet — press "Exit Wizard" if prompted
    3. 
                
  3. Purchase a Premium subscription (~$5/mo) — activation banner appears at the top of the dashboard
    4. 
                
  4. Go to Setup in the top bar → click "New Account Wizard" → select Docker when asked where to integrate playit
    5. 
                
  5. Continue to Agent Setup below
    6. 
              

            


            
            

              Agent Setup
              
    
                
  1. In the playit.gg dashboard, go to Agents → Add Agent
    2. 
                
  2. Name your agent (e.g. StardropHost-1) then press Next
    3. 
                
  3. A setup page will appear — copy the full docker compose snippet shown on the page
    
                  *You may need to click Exit Wizard if the screen doesn't refresh after this step
    4. 
                
  4. Paste the copied compose snippet directly into the Compose Entry card below and click Apply & Connect
    5. 
                
  5. The agent will show Connected in the playit.gg Agents tab once running
    6. 
                
  6. Continue to Tunnel Setup below
    7. 
              

            


            
            

              Tunnel Setup
              
    
                
  1. Click the Tunnels tab on playit.gg → click Create Tunnel
    2. 
                
  2. Name it (e.g. Stardrop-Share), select type UDP
    
                  Set port count to 1
    3. 
                
  3. Give it a description (e.g. Stardew Valley), then complete the usage confirmation
    4. 
                
  4. For Public End Point — select the region closest to you *Premium gives access to more regions
    5. 
                
  5. For Local IP & Port: copy your server's LAN IP from the StardropHost dashboard home page and enter port 24642 for the game
    6.

    7. 
                
  7. Press Next, select the agent created in the Agent Setup above, then click Create Tunnel
    8. 
                
  8. Your tunnel is now active. In the Tunnels tab you will see:
    
                  
                    Domain: yourname.playit.plus
    
                    IP & Port: 12.34.56.78:12345
                      
                  Share the IP & Port with your friends — they enter it in Stardew Valley's co-op join screen to connect from anywhere.
                
    9. 
              

              
*To use StardropHost Dashboard remotely: See Remote Dashboard Setup below — requires a TCP tunnel (Premium). No proxy server support yet.

            


            
            

              Remote Dashboard Setup TCP — Premium required
              

                The StardropHost dashboard is a web app served over HTTP. To access it remotely via playit.gg you need a TCP tunnel (Premium only — free tier is UDP only).
                No proxy server is required or supported yet.
              

              
    
                
  1. In playit.gg → Tunnels → Create Tunnel
    2. 
                
  2. Name it (e.g. Stardrop-Dashboard), select type TCP, port count 1
    3. 
                
  3. For Local IP & Port: use your server's LAN IP and port 18642 (the dashboard port)
    4. 
                
  4. For Public End Point — select the region closest to you
    5. 
                
  5. Select the agent created in Agent Setup, then click Create Tunnel
    6. 
                
  6. Your tunnel will show a domain and port — access the dashboard from anywhere at:
    
                  
                    http://yourname.playit.plus:PORT
                  
                
    7. 
                
  7. Log in with your StardropHost panel username and password
    8. 
              

              

                ⚠ HTTP only — no HTTPS yet. Do not expose sensitive credentials over an untrusted network. HTTPS via Certbot is a planned future feature.
              

            


            
            

              Docker Compose Example — playit.gg
              

                Copy the docker compose snippet from the playit.gg setup wizard — it will look like this but with your assigned SECRET_KEY.
              

              
services:
  playit:
    image: ghcr.io/playit-cloud/playit-agent:0.17
    network_mode: host
    restart: unless-stopped
    environment:
      - SECRET_KEY=YOUR_SECRET_KEY

              Advanced Docker Config
              ADVANCED
            

            

              

                For services that need to run alongside StardropHost but aren't tunnel providers — for example Certbot (HTTPS), a reverse proxy, or other infrastructure tools.
                These are added directly to your docker-compose.yml via SSH on your host machine.
              

              

                ⚠ Advanced — modifying docker-compose.yml incorrectly can break your stack. Take a backup first.
              

              

                Advanced docker features coming to the dashboard soon — no more SSH required.
              


              

                HTTPS with Certbot (Let's Encrypt)
                

                  Certbot obtains a free TLS certificate from Let's Encrypt so the dashboard can be served over HTTPS.
                  Requires a domain name pointed at your server's public IP and port 80 open for the ACME challenge.
                

                
    
                  
  1. Point your domain's DNS A record at your server's public IP
    2. 
                  
  2. Open port 80 on your VM/firewall (needed for certificate issuance only)
    3. 
                  
  3. SSH into your host and add a Certbot service to docker-compose.yml:
                    
      certbot:
    image: certbot/certbot
    volumes:
      - ./data/certbot/conf:/etc/letsencrypt
      - ./data/certbot/www:/var/www/certbot
    entrypoint: >
      certbot certonly --webroot
      --webroot-path=/var/www/certbot
      --email you@example.com
      --agree-tos --no-eff-email
      -d yourdomain.com
    
                  
    4. 
                  
  4. Run docker compose up certbot to obtain the certificate
    5. 
                  
  5. Add a reverse proxy (e.g. Nginx, Caddy) to serve the dashboard over HTTPS using the certificate in ./data/certbot/conf/
    6. 
                

                

                  Full HTTPS + reverse proxy setup guide coming in a future update.
